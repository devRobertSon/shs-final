# 과학고 대비 학습 포털

두 학원 약 20명의 학생·학부모가 **본인의 퀴즈 점수, 반 평균, 점수 추이 그래프, 숙제 체크,
주간 리포트, 공지사항, 자료실(PDF), 출석·진도**를 확인할 수 있는 무료 웹사이트입니다.

- **서버·비용 없음** — public GitHub 저장소 + GitHub Pages만 사용합니다. Actions/워크플로도 필요 없습니다.
- **저장소를 열어봐도 학생 정보가 보이지 않습니다** — 모든 학생 데이터(이름·점수·리포트)는
  브라우저에서 암호화된 뒤 커밋됩니다. 저장소에는 의미 없는 암호문과 무작위 파일명만 올라갑니다.
- 학생·학부모는 **접속 코드 한 번 입력**으로 본인 데이터만 복호화해서 봅니다.
- 학원별로 공지·자료실이 **완전히 분리**됩니다 (다른 학원 것은 열 수 없음).
- **선생님 열람 코드**(학원별): 같은 로그인 화면에서 이 코드로 들어가면 그 학원의
  출석 현황, 숙제 체크, 학생별 성적표, 퀴즈 점수 분포, 개별 리포트, 공지를 볼 수 있습니다. 편집은 불가능하며,
  관리는 마스터 비밀번호를 가진 관리자만 할 수 있습니다. (학생 관리 탭에서 발급)

> 지금 저장소에는 동작 확인용 **가상 학생 샘플 데이터**가 들어 있습니다.
> 테스트 코드는 `dev/SAMPLE.md` 참고. 실제 사용 시 초기 설정(아래 1-③)을 하면 교체됩니다.

---

## 1. 처음 한 번만 하면 되는 설정

### ① GitHub Pages 켜기

1. 이 저장소(public)의 **Settings → Pages**로 이동합니다.
2. **Build and deployment → Source**를 `Deploy from a branch`로 두고,
   **Branch**를 `main` / `/ (root)`로 선택 후 **Save**.
3. 1~2분 뒤 `https://<사용자명>.github.io/<저장소이름>/` 주소가 생깁니다.

### ② 관리 페이지 접속

`https://<사용자명>.github.io/<저장소이름>/admin.html` 을 엽니다.
(관리 페이지 주소는 학생에게 알려줘도 마스터 비밀번호 없이는 아무것도 볼 수 없지만, 알릴 필요는 없습니다.)

### ③ 초기 설정 마법사

1. **마스터 비밀번호**(12자 이상, 문장형 추천)를 정합니다.
   ⚠️ 이 비밀번호를 잊으면 복구할 수 없습니다. 반드시 안전한 곳에 적어 두세요.
2. 학원 이름들과 학생 명단을 입력하면 학생별 **접속 코드**가 자동 생성됩니다.
3. **학생 관리 탭 → 전체 코드 카드 인쇄**로 A4 코드 카드를 인쇄해 학생/학부모에게 나눠 줍니다.
4. **발행 탭**에서 사이트에 올립니다(아래 2번 참고).

### ④ PAT(개인 접근 토큰) 만들기 — 자동 발행용

발행을 클릭 한 번으로 하려면 GitHub 토큰이 필요합니다.

1. GitHub 우측 상단 프로필 → **Settings → Developer settings →
   Personal access tokens → Fine-grained tokens → Generate new token**
2. 설정:
   - **Repository access**: `Only select repositories` → 이 저장소만 선택
   - **Permissions → Repository permissions → Contents**: `Read and write`
   - 그 외 권한은 모두 No access, 만료는 90일 추천 (만료되면 새로 만들면 됩니다)
3. 생성된 `github_pat_…` 문자열을 발행 탭에 붙여넣습니다.
   토큰은 **이 브라우저에만** 저장되며 저장소에는 절대 올라가지 않습니다.
   공용 컴퓨터에서는 "이 브라우저에 토큰 저장"을 체크하지 마세요.

> 토큰 없이도 사용할 수 있습니다: 발행 탭의 **ZIP 다운로드** → 압축 해제 →
> GitHub 저장소 페이지에서 **Add file → Upload files** → `data` 폴더를 통째로 끌어다 놓고
> **Commit changes**. (폴더째 드래그해야 경로가 유지됩니다)

### ⑤ (선택) 접속 통계 켜기

관리 페이지 **발행 탭**에서 포털 접속(로그인) 횟수를 볼 수 있습니다.
GitHub 릴리스의 다운로드 카운트를 이용하므로 **외부 분석 서비스가 없고,
GitHub 밖으로는 아무 정보도 나가지 않습니다.** 한 번만 설정하면 됩니다:

1. 저장소 페이지 → 오른쪽 **Releases** → **Draft a new release**
2. **Choose a tag**에 `visit-counter` 입력 → "Create new tag" 클릭. 제목은 아무거나 (예: 접속 통계)
3. 파일 첨부 칸에 저장소의 `dev/visit-student.bin`, `dev/visit-teacher.bin` 두 파일을
   내려받아 그대로 첨부 (또는 같은 이름의 아무 작은 파일 두 개를 만들어 첨부)
4. **Publish release** → 다음 로그인부터 집계되고, 발행 탭 '접속 통계' 카드에서 확인

- 집계 방식: 로그인 성공 시 브라우저 탭 기준 1회 — **고유 인원이 아니라 접속 횟수**입니다.
- 학생·학부모·선생님 화면에는 아무것도 표시되지 않습니다.
- 릴리스를 만들지 않으면 이 기능은 조용히 꺼져 있습니다 (아무 영향 없음).

## 2. 매주 하는 일 (5~10분)

1. `admin.html` 접속 → 마스터 비밀번호 입력
2. **주차 관리**에서 이번 주차 추가 (수업일 입력)
3. **점수 입력**: 엑셀에서 `이름 ⇥ 점수` 두 열을 복사해 붙여넣기 → 저장.
   결석 등으로 퀴즈를 안 본 학생은 **[미응시]** 버튼(또는 점수 칸에 `결`)으로 표시 —
   평균·응시 인원에서 제외되고, 보고서에서 '미입력' 경고 대신 미응시로 표기됩니다.
4. **숙제 체크**: 숙제 항목 입력 → 학생별 체크 → 필요하면 **카톡용 숙제 목록 복사** 버튼으로
   숙제 목록을 복사해 단체 카톡방에 붙여넣기.
   결석한 학생은 칸을 **◌(확인 전)**으로 두면 완료율에서 빠지며,
   다음 수업에서 확인한 뒤 ✓ 또는 빈칸으로 바꿉니다.
5. **출석 / 리포트 / 공지 / 자료실 / 진도** 입력
6. **발행** 탭 → 🚀 발행 → 1~2분 뒤 사이트 반영

학생/학부모는 코드 카드의 주소로 접속해 코드를 입력하면
(자동 로그인을 체크하면 다음부터 입력 없이) 본인 데이터를 봅니다.
마지막 접속 이후 새로 올라온 공지·퀴즈 점수·리포트가 있으면 해당 탭에
빨간 점(●)이 표시되고, 탭을 열어보면 사라집니다.

퀴즈에 대한 질문은 수업 카톡방/개인 카톡으로 받습니다 (사이트에는 질문 기능이 없습니다).

## 3. 보안 모델 — 무엇이 보호되고, 무엇이 한계인가

**저장소를 열어 본 사람이 볼 수 있는 것**: 무작위 파일명, 암호문, salt 값, 학생 수, 커밋 시각.
이름·점수·리포트·코드 등 의미 있는 정보는 전혀 볼 수 없습니다.

- 학생 접속 코드(10자리, 49비트 상당)에서 PBKDF2-SHA256(31만 회) → HKDF로
  **파일 ID**와 **AES-256-GCM 키**를 분리 유도합니다. 파일명으로 코드를 역산할 수 없고,
  무차별 대입도 현실적으로 어렵습니다.
- 학원 공지·자료실은 학원별 무작위 키로 암호화되고, 이 키는 소속 학생의 암호문 안에만 들어 있습니다.
  → A학원 학생은 B학원 데이터를 열 수 없습니다.
- 관리용 명부(roster)는 마스터 비밀번호(60만 회 반복)로 암호화됩니다.

**한계(알고 계세요)**:
- 학생이 자기 코드를 남에게 알려주면 **그 학생의 데이터와 소속 학원의 공지·자료**까지 보입니다.
  → 학생 관리 탭에서 **코드 재발급**으로 대처.
- 학생을 **삭제**하면 그 학생이 알던 학원 키도 자동으로 교체됩니다(공지·자료 재암호화).
  삭제 후 반드시 발행하세요.
- git 히스토리에는 과거 암호문이 남지만, 같은 키로 보호되므로 문제되지 않습니다.
- 마스터 비밀번호를 잊으면 데이터를 열 수 없습니다. **발행 탭의 백업 다운로드**를
  주기적으로 해 두면, 로그인 화면의 "백업 파일로 복원"으로 새 비밀번호를 설정할 수 있습니다.

## 4. 문제 해결

| 증상 | 해결 |
|------|------|
| 마스터 비밀번호 분실 | 백업 JSON이 있으면: 로그인 화면 → **백업 파일로 복원** → 새 비밀번호 → 발행. 백업이 없으면 초기 설정을 다시 하고 코드 카드를 재배부해야 합니다. |
| 학생이 코드 분실 | 학생 관리 → 해당 학생 코드가 그대로 보이므로 다시 알려주거나, 유출이 의심되면 **코드 재발급** 후 발행. |
| 발행했는데 사이트가 안 바뀜 | Pages 반영에 1~2분 걸립니다. 그래도 안 되면 새로고침(모바일: 당겨서 새로고침). 이 앱은 캐시를 우회하도록 만들어져 있어 보통은 바로 보입니다. |
| Actions의 "pages build and deployment"가 실패 (`Deployment failed, try again later`) | GitHub 쪽 일시 오류로, 파일 문제가 아닙니다. 저장소 **Actions 탭 → 실패한 실행 → Re-run failed jobs**를 누르면 이미 만들어진 빌드로 30초 내 재배포됩니다. 다음 커밋이나 발행 때 자동으로 함께 반영되기도 합니다. |
| 실수로 이전 상태를 발행해 공지·점수가 사라짐 | 오래 열려 있던 관리자 탭에서 발행하면 그 사이 다른 곳에서 발행한 내용이 덮어써질 수 있습니다 (발행 시 "다른 곳에서 발행된 뒤입니다" 경고가 뜨면 취소하고 새로고침 후 다시 작업하세요). 이미 덮어썼다면 `사이트주소/dev/recover.html`에서 과거 발행 커밋의 공지·점수·진도를 마스터 비밀번호로 열어 보고, 관리 페이지에 다시 입력해 발행하면 됩니다. |
| 발행 시 401/403 오류 | PAT가 만료됐거나 권한이 부족합니다. 1-④를 참고해 다시 만드세요. |
| 발행 시 409/422 오류 | 다른 곳에서 저장소가 바뀌었습니다. 페이지 새로고침 후 다시 로그인해 재시도. |
| "보안 컨텍스트가 아닙니다" | `file://`로 열면 동작하지 않습니다. GitHub Pages 주소나 `python3 -m http.server`(localhost)로 여세요. |
| ZIP 수동 업로드를 썼는데 옛 파일이 남음 | 웹 업로드는 파일 삭제를 못 합니다. 남은 파일은 어차피 읽을 수 없는 암호문이라 무해하지만, 정리하려면 GitHub에서 직접 삭제하거나 자동 발행(PAT)을 쓰세요. |

## 5. 제한 사항

- 자료실 PDF는 **25MB 이하 권장** (GitHub 파일 한도 100MB, 저장소 권장 총량 ~1GB).
- 관리 페이지는 선생님 1명이 쓰는 것을 가정합니다. 두 기기에서 동시에 편집하지 마세요.
  (이 화면을 연 뒤 다른 곳에서 발행된 적이 있으면 발행 직전에 경고가 표시됩니다.)
- 편집 내용은 **발행 전까지 브라우저 메모리에만** 있습니다. 발행하기 전에 창을 닫으면 사라집니다.
  오래 열어 둔 탭에서 이어서 작업하지 말고, 새로고침 후 다시 로그인해 최신 상태에서 작업하세요.

## 6. 개발자 참고

- 순수 HTML/CSS/JS (빌드 없음). 로컬 실행: `python3 -m http.server 8000` → `http://localhost:8000`
- 암호화 셀프테스트: `http://localhost:8000/dev/test.html`
- 샘플 데이터 재생성: `node dev/make-sample.mjs` (코드는 `dev/SAMPLE.md`에 기록)
- 데이터 구조와 암호화 설계는 `js/crypto.js`, `js/store.js` 상단 주석 참고
